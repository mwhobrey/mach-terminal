## What's Working (shipped + has guardrails)

### Core terminal
- Multi-session PTY lifecycle: spawn / write / resize / close, reader threads, bounded output chunking + monotonic sequencing, runtime counters. (`session_manager.rs`)
- Output pacing: frontend RAF batching + per-frame byte budget + bounded scrollback; sequence anomaly detection (duplicate/gap/resync).
- Tabs + split workspace: **binary split tree** per tab group (up to **6 panes**), draggable resize handles, unified **group composer** (Operator mode), **pane focus** vs **composer target** (independent; see below), broadcast mode (`off` | `once` | `sticky`, TER-19), command palette + keymap.
- **Pane focus vs target:** **Focus** (`Ctrl+Alt+N` Win · `Alt+N` Mac) = keyboard/terminal UI + AI/ops context (cyan left bar). **Target** (`Ctrl+Alt+Shift+N` Win · `Alt+Shift+N` Mac) = group composer routes Enter/exit/completion (purple bottom bar). Click pane → focus; composer pill → target only. Closing a pane syncs both to the survivor.
- Exit lifecycle: exit overlay (`Session stopped/closed/error`, `Exited with code N`), tab status dots/tooltips, restart-in-same-pane, batch close/restart-all-exited.
- Composer-first input: completion (cwd-aware path + PATH/builtin command names), prediction, bounded history browsing, output-scroll paging, assist metrics.
- Workspace layout persistence (`workspace_layout.json`) with legacy `localStorage` migration + ghost-session reconciliation; layout also carries `sessions[]` (`RestorableSession`: shell/**args**/cwd/custom name/**chatKey**) so tabs are respawned + renamed on next launch; cold restart respawns per-tab shell+cwd+args; AI chat hydrates from localStorage via stable `chatKey`.
- Command history (`command_history.json`) with corruption recovery/backup + one-time toast; query/replay.
- Link safety (http/https + safe file only; OSC 8 hyperlinks; unsafe schemes are no-ops), safe-paste guard for risky multiline payloads, BEL visual flash, context menu with viewport clamp.

### Shell integration
- OSC 7 (live cwd → restart cwd) and OSC 133 (command markers → status hint), both opt-in via rc snippets.
- Cross-shell (pwsh/bash/zsh) hook install/remove/status/backup/restore via strategy-map dispatch; health diagnostics (`healthy/stale/missing/error`); pwsh profile override.
- `shell_context_snapshot` (elevation, git branch/short-stat).

### AI / providers (optional, off by default)
- Executable adapters for **openai, anthropic, ollama, custom-openai** (`provider_host.rs`).
- OS-keychain API key storage (`provider_secrets.rs`); endpoint validation (http/https); per-provider model routing.
- AI explain/fix/freeform with bounded prompt context (cwd/shell/≤6000-char scrollback), **ops-rail AI chat tab**, opt-in + in-flight gating, stale-response supersession guard.
- **AI internals v2:** multi-turn `history[]` on execute, `system_prompt` + `ai_context_budget_chars` in routing, chat persistence via `chatKey` + localStorage, merged attachment/scrollback excerpts.
- **AI read-only tools:** `list_command_runs` / `get_command_output` resolved on the frontend from ops-rail ledger + scrollback; OpenAI-compatible, Ollama, and Anthropic providers run a native tool loop (`aiToolRunner.ts`, max 4 rounds); Settings toggle **Enable AI command-log tools** (`aiBehaviorSettings.enableAiTools`, default on).

### Plugins
- Capability-scoped `PluginHost` with reason-coded policy decisions, grants snapshot, execution + telemetry metrics. Registry: `history-tools`/`session-restore` (alpha), `provider-router` (planned). Surfaced via Settings plugin demo only.

### Platform / quality
- Custom title bar (decorations off), diagnostics snapshot (debug builds), telemetry (JSON logs + optional OTLP).
- CI matrix build/test, release smoke on ubuntu matrix leg (debug deb, warmed runner), release preflight gates, nightly burn-in + recalibrated threshold gates, stability signoff report, security baseline (npm + cargo audit).
- Extensive automated smoke coverage for terminal interaction contracts (palette, find, link safety, exit lifecycle, pane focus/follow, provider UX, composer, paste/BEL).

- **Phase 1 hot-path transport (raw-bytes Channel).** Migrated `pty-output` off the Tauri **event system** (which JSON-serializes every payload and is explicitly "not designed for low latency or high throughput" per Tauri docs) onto a single long-lived `tauri::ipc::Channel<Response>` carrying raw bytes. The webview registers it once at mount via a new `pty_subscribe_output` command (`SessionManager.output_channel: Arc<Mutex<Option<Channel<Response>>>>`); reader threads stream framed chunks (`[u16 LE id_len][session_id][chunk utf8]`). This **preserved the clean central-listener design** (one handler, no per-spawn rewiring) and dodged the per-session-channel spawn/first-byte race — the lifetime constraint (`Channel<&[u8]>` can't move into the `'static` reader thread) meant an owned message type was required either way, so the single-channel shape won. The bridge (`onPtyOutput` in `terminal.ts`, consumed by `usePtyOutputStream` — same `PtyOutputEvent` shape) parses the frame and synthesizes a per-session monotonic `sequence` (channel delivery is ordered), so the downstream coalesce/anomaly contract is untouched. Removed the `PtyOutputEvent` Rust emit + `PTY_OUTPUT_EVENT` const. Added a `pty_byte_pipeline_round_trips_and_meets_throughput_floor` bench (decode+chunk **baseline 72 MiB/s debug**, 16 MiB round-tripped byte-for-byte, zero spurious U+FFFD, 50 MiB/s floor). Gate: types, UX 413, smoke 36, cargo `--lib` 87, invoke-smoke. **Follow-up fix (multi-tab/resume fuckery):** two races exposed by the faster channel — (1) boot `loadCapabilities` spawned/restored PTYs before `pty_subscribe_output` completed (backend dropped pre-subscribe output); fixed with `ensurePtyOutputSubscribed()` awaited before any spawn + a 512-frame pre-handler backlog replayed when `onPtyOutput` wires the handler. (2) `pty-lifecycle` `running` cleared `pendingOutputRef` but channel output often arrives *before* that event, wiping shell prompts on cold multi-tab restore; fixed by only resetting sequence baseline on `running`, not pending chunks. `clearSessionFromUiState` now prunes `pendingOutputRef`/`lastSequenceRef`. Docs: `runtime-contracts.md` updated. **Still requires manual `tauri dev` smoke** for live channel round-trip.
- **Phase 0 hot-path fixes (speed/correctness foundation).** Two grounded fixes against the "speed-first" North Star: (1) **WebGL renderer** — `TerminalSurface` now loads `@xterm/addon-webgl` after `terminal.open()` (xterm v6's default DOM renderer was the slowest path; no GPU addon was present), with graceful fallback: `onContextLoss` + init `try/catch` dispose the addon and revert to the DOM renderer; addon disposed in teardown before `terminal.dispose()`. (2) **Streaming UTF-8 decode** — the reader thread replaced per-read `String::from_utf8_lossy` (which mangled any multibyte codepoint split across an 8 KB `read()` boundary into U+FFFD) with `decode_utf8_streaming`, carrying the incomplete trailing sequence (<4 bytes) into the next read; genuine invalid bytes still become a single U+FFFD; OSC 7/133 parsers still see raw bytes. Empty-output reads no longer emit a phantom chunk. 4 new Rust unit tests (split 3-byte/4-byte sequences, invalid-byte replacement, valid passthrough). Gate: types, UX 413, smoke 36, cargo `--lib session_manager` 11. **Deferred (was mischaracterized as urgent):** the `MAX_PENDING_CHUNKS` drop is effectively dead code at 8 KB reads (≤4 chunks vs 64 cap) — real flow control / binary `Channel` transport is Phase 1, not a correctness fix.
